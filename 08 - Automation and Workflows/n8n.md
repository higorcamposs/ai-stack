# n8n

## O que é
n8n é uma plataforma de automação orientada a workflows, com abordagem visual e extensível, usada para integrar sistemas, eventos e tarefas.

## Para que serve
Serve para montar automações, integrações e pipelines operacionais sem precisar escrever toda a orquestração do zero.

## Onde entra na stack
Entra na camada de automação e workflows, próxima de integrações, operações recorrentes e gatilhos que podem acionar agentes ou serviços de IA.

## Quando faz sentido usar
- Quando há muitos serviços para integrar.
- Quando o fluxo é mais operacional do que cognitivo.
- Quando o time quer velocidade para automatizar processos internos.

## Quando não faz sentido usar
- Quando o fluxo exige lógica muito sofisticada e código pesado.
- Quando a automação precisa de governança que a modelagem visual não deixa clara.
- Quando o time assume que workflow visual substitui desenho de arquitetura.

## Pontos fortes
- Muito útil para integração entre sistemas.
- Acelera automações de negócio e operações internas.
- Pode servir como ponte entre IA, banco, notificações e serviços externos.

## Limitações
- A licença não é uma licença open source clássica; vale ler com cuidado.
- Fluxos grandes podem ficar difíceis de manter.
- Nem toda automação agentic se encaixa bem no modelo visual do n8n.

## Alternativas
- Zapier
- Make
- Workflows próprios com código

## Caso de uso real
Uma equipe pode usar n8n para receber eventos de CRM, enriquecer dados com IA, registrar tudo em sistemas internos e disparar notificações para revisão humana.

## Minha leitura
O n8n parece excelente para operacionalizar a stack, desde que o time não confunda automação visual com arquitetura completa. Ele resolve muito bem a cola entre sistemas.
