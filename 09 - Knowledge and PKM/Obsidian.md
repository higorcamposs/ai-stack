# Obsidian

## O que é
Obsidian é um editor Markdown e aplicativo de base de conhecimento focado em arquivos locais, links internos e organização de notas conectadas.

## Para que serve
Serve para criar um sistema pessoal ou de equipe para estudar, documentar, relacionar ideias e manter conhecimento em arquivos duráveis.

## Onde entra na stack
Entra na camada de [[09 - Knowledge and PKM/Obsidian|conhecimento e PKM]], funcionando como base para notas, mapas e documentação de ferramentas, conceitos e decisões.

## Quando faz sentido usar
- Quando o usuário valoriza arquivos Markdown locais.
- Quando links entre notas fazem parte do método de estudo.
- Quando a base precisa continuar legível fora do aplicativo, inclusive no GitHub.

## Quando não faz sentido usar
- Quando a equipe precisa de colaboração simultânea forte por padrão.
- Quando o método depende menos de notas e mais de banco estruturado.
- Quando o usuário não quer cuidar da própria estrutura de conhecimento.

## Pontos fortes
- Arquivos locais e portáveis.
- Boa experiência para conhecimento conectado.
- Funciona bem para documentação pessoal, pesquisa e estudo.
- Forte alinhamento com Git e Markdown simples.

## Limitações
- O aplicativo principal não é open source.
- A qualidade do sistema depende muito da disciplina de organização.
- Sem critério, o vault vira acúmulo de notas sem síntese.

## Alternativas
- Logseq
- Notion
- Sistemas próprios baseados em Markdown

## Caso de uso real
Este próprio cofre é um caso de uso: organizar estudo de ferramentas de IA, comparações, roadmaps e casos reais em Markdown legível no Obsidian e no GitHub.

## Minha leitura
Obsidian é menos uma solução pronta de conhecimento e mais um ambiente para construir método. O valor real aparece quando há clareza de estrutura, revisão e síntese.
