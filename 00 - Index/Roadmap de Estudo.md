# Roadmap de Estudo

## Objetivo

Construir entendimento progressivo sobre ferramentas de IA e evoluir de uso pontual para desenho de sistemas.

## Etapa 1: Base conceitual

- Estudar LLMs, contexto, tokens, embeddings e limites de modelos
- Entender diferença entre chat, agente, workflow e automação
- Criar notas em `01 - Foundations` e `02 - Models`

## Etapa 2: Ferramentas centrais

- Mapear interfaces de chat e agentes de código
- Comparar frameworks de agentes
- Registrar critérios práticos de escolha

## Etapa 3: Memória e recuperação

- Estudar RAG, indexação, chunking e reranking
- Entender memória de sessão, memória persistente e contexto externo
- Consolidar padrões em `06 - RAG and Memory`
- Ver [[13 - Real Use Cases/RAG para documentação técnica|RAG para documentação técnica]]
- Ver [[13 - Real Use Cases/Memória operacional com Qdrant + incidentes|Memória operacional com Qdrant + incidentes]]

## Etapa 4: Operação e automação

- Explorar workflows, integrações e execução local
- Documentar pipelines simples e compostos
- Relacionar custo, controle, velocidade e manutenção
- Ver [[13 - Real Use Cases/Workflows de automação com n8n + IA|Workflows de automação com n8n + IA]]
- Ver [[13 - Real Use Cases/Stack mínima para estudar agentes localmente|Stack mínima para estudar agentes localmente]]

## Etapa 5: Observabilidade e qualidade

- Definir como medir qualidade, latência, custo e confiabilidade
- Documentar tracing, avaliação e monitoramento
- Criar comparações orientadas a decisão
- Ver [[13 - Real Use Cases/Observabilidade de aplicações LLM com Langfuse + OpenTelemetry|Observabilidade de aplicações LLM com Langfuse + OpenTelemetry]]
