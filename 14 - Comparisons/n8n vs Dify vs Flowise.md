# n8n vs Dify vs Flowise

## Objetivo da comparação
Comparar [[08 - Automation and Workflows/n8n|n8n]], Dify e Flowise como plataformas visuais para montar fluxos com IA e automação.

## Semelhanças
- As três reduzem a necessidade de escrever toda a orquestração na mão.
- As três ajudam a acelerar protótipos e integrações.
- As três podem ser usadas para conectar modelos, dados, APIs e lógica de fluxo.

## Diferenças
- [[08 - Automation and Workflows/n8n|n8n]] é mais forte como automação operacional geral entre sistemas.
- Dify é mais forte como plataforma integrada para aplicações com LLM, com workflow, RAG, agentes e camada mais próxima de produto.
- Flowise é mais forte como builder visual de fluxos e agentes para experimentação rápida, mas exige cuidado extra de operação e segurança.
- n8n vem da lógica de integração.
- Dify vem da lógica de plataforma de app com IA.
- Flowise vem da lógica de composição visual de LLM apps e agentes.

## Melhor para cada caso
- n8n: para automação entre sistemas, eventos, bancos, notificações e processos operacionais.
- Dify: para times que querem uma plataforma visual mais completa para apps com IA, especialmente com RAG e chat.
- Flowise: para protótipos rápidos de fluxos e agentes visuais, com time técnico atento à operação.

## Pontos de decisão
- Se o problema principal é automação de negócio, eu começaria por [[08 - Automation and Workflows/n8n|n8n]].
- Se o problema principal é construir um produto com IA de forma visual, eu olharia primeiro para Dify.
- Se o time quer máxima liberdade para experimentar fluxos agentic rapidamente, Flowise pode ser útil, mas merece atenção redobrada em segurança e exposição pública.
- Se a equipe confunde prototipagem visual com prontidão de produção, qualquer uma das três pode ser mal escolhida.

## O que eu escolheria e por quê
Eu escolheria [[08 - Automation and Workflows/n8n|n8n]] para automação operacional, Dify para plataforma visual mais completa de app com IA e Flowise para experimentação controlada. Se eu tivesse que escolher apenas uma para adoção de equipe com objetivo claro de negócio, provavelmente começaria por n8n ou Dify, não por Flowise.
