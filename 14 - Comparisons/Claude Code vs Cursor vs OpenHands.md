# Claude Code vs Cursor vs OpenHands

## Objetivo da comparação
Comparar [[05 - Coding Agents/Claude Code|Claude Code]], [[05 - Coding Agents/Cursor|Cursor]] e [[05 - Coding Agents/OpenHands|OpenHands]] como abordagens para desenvolvimento assistido por agentes.

## Semelhanças
- Os três tentam aumentar produtividade em tarefas de engenharia.
- Os três operam sobre contexto de código, tarefas e instruções em linguagem natural.
- Os três fazem mais sentido com revisão humana e processo mínimo de validação.

## Diferenças
- Claude Code é mais forte para fluxo centrado em terminal e repositório local.
- Cursor é mais forte para fluxo centrado em editor, com experiência de produto mais integrada.
- OpenHands é mais interessante para estudar ou operar uma abordagem open source de agente de software.
- Cursor tende a ser o mais polido como produto.
- Claude Code tende a ser o mais direto para quem já pensa em shell, repositório e execução.
- OpenHands tende a exigir mais maturidade técnica e tolerância a operação.

## Melhor para cada caso
- Claude Code: para quem trabalha pesado em terminal e quer um agente pragmático no fluxo real.
- Cursor: para quem quer a melhor experiência integrada dentro da IDE.
- OpenHands: para quem quer estudar, experimentar ou operar uma stack open source de coding agents.

## Pontos de decisão
- Se a equipe é muito orientada a terminal, Claude Code tende a encaixar melhor.
- Se a equipe vive dentro do editor, Cursor tende a gerar adoção mais rápida.
- Se a prioridade é controle, inspeção e self-hosting, OpenHands ganha relevância.
- Se o objetivo é produtividade imediata com menos atrito, Cursor e Claude Code ficam à frente.
- Se o time ainda não tem cultura de revisão, nenhum deles deve ser tratado como piloto automático.

## O que eu escolheria e por quê
Hoje eu escolheria [[05 - Coding Agents/Claude Code|Claude Code]] para fluxo técnico profundo em terminal, [[05 - Coding Agents/Cursor|Cursor]] para produtividade ampla de equipe em editor e [[05 - Coding Agents/OpenHands|OpenHands]] para laboratório ou stack open source. Se eu tivesse que escolher só um para trabalho individual intenso, escolheria Claude Code. Para equipe querendo adoção rápida, Cursor. Para pesquisa e autonomia de stack, OpenHands.
