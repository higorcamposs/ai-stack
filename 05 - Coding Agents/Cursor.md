# Cursor

## O que é
Cursor é um editor de código com IA embutida, orientado a geração, edição, compreensão de base e agentes em ambiente de desenvolvimento.

## Para que serve
Serve para acelerar tarefas de desenvolvimento dentro da interface do editor, incluindo chat sobre código, edição por instrução e agentes em background.

## Onde entra na stack
Entra na camada de ferramentas de desenvolvimento com IA, entre IDE assistida e [[05 - Coding Agents/Claude Code|agente de código]].

## Quando faz sentido usar
- Quando a equipe quer IA integrada ao editor.
- Quando há benefício em misturar edição local com agentes remotos.
- Quando o fluxo de trabalho gira mais em torno da IDE do que do terminal.

## Quando não faz sentido usar
- Quando a organização prefere stack open source ou self-hosted.
- Quando a equipe trabalha em ambientes muito restritos.
- Quando o ganho sobre um editor comum com extensões não compensa o custo.

## Pontos fortes
- Experiência de produto muito refinada para desenvolvimento assistido.
- Une autocomplete, chat, edição e agentes em uma interface só.
- Background agents ampliam o escopo para tarefas assíncronas.

## Limitações
- Dependência de fornecedor e política comercial.
- Recursos avançados podem envolver execução remota e retenção temporária de dados.
- Nem todo time precisa da camada completa de agentes para ganhar produtividade.

## Caso de uso real
Um desenvolvedor pode usar Cursor para explorar uma base desconhecida, pedir alterações em linguagem natural, revisar trechos e delegar tarefas secundárias a agentes em background.

## Minha leitura
Cursor é relevante porque mostra a convergência entre editor, copiloto e agente. Para estudo, vale separar o que é ganho real de fluxo e o que é apenas conveniência de interface.
